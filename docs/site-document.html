1. Overview

This document summarizes the Event Planner demo website. The site is built with Next.js and React, and includes sections for exploring events, packages, booking, calendar availability, testimonials, and a gallery. The layout is component-driven and responsive.

2. Navigation

The site uses a top navigation bar with anchor-style navigation that leverages react-scroll's Element API. Sections are registered with names such as hero, events, packages, and gallery. Smooth scrolling is enabled and the nav highlights the active item when clicked.

3. Hero

The hero section provides the main visual introduction and may include a video background (component FixedVideoBackground), a title and a call to action. It's wrapped inside a ScrollElement with name hero so navigation targets it directly.

4. Events

Events are presented using an events grid and a 3D carousel. The components involved include EventsGrid and events-3d-carousel. Event cards show images, titles and short descriptions highlighting featured events.

5. Packages

Packages present different pricing and feature sets for event planning. The Packages component is responsible for layout and package cards.

6. Booking

The booking flow includes a calendar view, a booking hero section and a form implemented in BookingForm. The form uses React Hook Form and Zod for validation. Availability is checked via the BookingCalendar component.

7. Calendar (Availability)

A dedicated calendar checks availability and surfaces open dates. It is intentionally full-screen and sits behind a semitransparent overlay to focus the user's attention on selection.

8. Testimonials

Client testimonials are displayed using the Testimonials component with cards and optional carousel navigation.

9. Gallery

The gallery displays a grid of images. The demo code includes a static image array used for the feed as fallback content. Below are a few sample images used in the site.

10. Contact

The contact section includes a map component and contact information so visitors can reach out to the events team. The contact map is implemented via ContactMap.

11. Dashboard

A preview dashboard component provides quick access to administrative interfaces and metrics. The demo includes a DashboardPreview component that illustrates the dashboard layout.
